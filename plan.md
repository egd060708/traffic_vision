# 实验四：复杂环境下基于机器视觉处理的车牌识别系统

## 总体策略：双轨结构

- **主实验（量化）**：在 CCPD（静态图片）上训练、调参、分场景评测，给出准确率/召回率等指标。
- **视频验证（定性，满足实验要求 2）**：模型训好后，用**自拍一段真实行车/停车视频**，演示视频流上的实时车牌识别，记录 FPS。
- 视频模块不依赖 CCPD（CCPD 是图片不是视频），自拍视频负责"实时识别"的落地证明。

## 技术方案决策

| 决策项 | 方案 |
|--------|------|
| 编程语言/框架 | Python + PyTorch + OpenCV + sklearn |
| 数据集 | CCPD（中国城市车牌开源数据集，静态图片） |
| 车牌检测 | YOLOv8-Pose（边界框 + 4 角点关键点联合检测，一次前向） |
| 车牌校正 | 4 角点透视变换（优于参考流程中的仿射变换） |
| 字符分割 | 投影分割法（二值化 + x/y 轴投影），带字符数校验 |
| 字符识别 | HOG 特征 + SVM（统一 65 类）；另加轻量 CNN 作对比组 |
| 视频识别 | 逐帧推理 + IoU 跟踪 + 跨帧多帧投票 |
| 数据规模 | 从 CCPD 采样 3-5 万张（base 2-3 万 + 各复杂子集各 2000-3000） |
| 硬件 | NVIDIA 3090（24GB 显存） |

## 改进点 / 创新点（评分重点，需在报告中显式论述）

1. **联合检测**：用 YOLOv8-Pose 一次性输出框 + 4 角点，相比"先检测框、再单独回归角点"的传统两段式，少一次前向、误差不累积。
2. **透视变换校正**：参考流程图给的是仿射变换，本方案用 4 角点透视变换，对倾斜/透视畸变的车牌校正更准。
3. **视频多帧投票**：同一车牌在连续帧的识别结果做多数票融合，把字符级 ~96% 的识别器在视频上拉到接近车牌级 100%（单图无法做到）。
4. **字符识别对比**：HOG+SVM（贴合"机器学习方法"考察）vs 轻量 CNN，给出准确率/速度 trade-off 分析。

## 字符识别预期与取舍（提前对齐预期）

- HOG+SVM 字符级准确率预计 ~96%；**车牌级（7 位全对）≈ 0.96⁷ ≈ 75%**，这是单图天花板。
- 拉高车牌级准确率的手段：视频多帧投票（见改进点 3）、识别后语言模型/规则校验、或切到 CNN。
- **65 类构成**：31 省份汉字 + 24 个字母（无 I、O，避免与 1、0 混淆）+ 10 数字 = 65。
- 主方案统一 65 类；可选**分位置建模**（第一位 31 类汉字、其余位 34 类），报告讨论取舍。

## 实施步骤

### Step 1：环境搭建与数据准备

1. 安装依赖：`ultralytics`, `opencv-python`, `scikit-learn`, `scikit-image`, `torch`, `matplotlib`, `tqdm`
2. 下载 CCPD 数据集
3. 编写 `prepare_data.py`：
   - 解析 CCPD 文件名中的标注信息（边界框、4 角点、车牌号、倾角、亮度）
   - 从各子集采样，总计 3-5 万张
   - 转换为 YOLOv8-Pose 格式（图片 + TXT 标注，按 8:2 划分训练/验证）
   - 生成 `data.yaml` 配置文件

**YOLOv8-Pose 标注格式（每行）：**
```
class cx cy w h  px1 py1 v1  px2 py2 v2  px3 py3 v3  px4 py4 v4
```
- `cx cy w h`：归一化的边界框中心坐标与宽高
- `px py v`：归一化的关键点坐标 + visibility；CCPD 4 角点全部可见，`v` 统一设为 **2**
- ⚠️ 少了 `v` 维度训练会报错，务必带

### Step 2：训练 YOLOv8-Pose 车牌检测器

1. 编写 `train_detector.py`：
   - 使用 `yolov8n-pose.pt` 预训练权重作为起点
   - 输入：YOLOv8-Pose 格式数据
   - 输出：边界框 + 4 角点关键点
   - 训练参数：epochs=50-100，batch_size 按 24GB 显存调整（预计 16-32）
2. 验证检测效果（mAP、关键点 OKS），保存最佳模型到 `models/license_plate_det.pt`

### Step 3：用 GT 角点生成字符数据集（关键：不经过检测模型）

1. 编写 `gen_chars.py`：
   - 直接用 CCPD 文件名里的 **GT 4 角点**做透视变换校正（不依赖检测模型推理，避免标签污染）
   - 灰度化 → 自适应二值化 → 水平投影去上下边框 → 垂直投影切字符
   - 根据 CCPD 标注的车牌号，按顺序给每个字符图分配标签
   - **字符数校验**：分割出的字符数 ≠ 7（蓝牌）→ 丢弃该样本，不进训练集
   - 按类别（65 类）保存到 `data/processed/chars/`
2. 统计每类样本数，类别不均衡时记录（报告分析用）

### Step 4：训练 HOG+SVM 字符分类器

1. 编写 `train_svm.py`：
   - 加载字符图，resize 到统一尺寸（如 20×40 或 32×64）
   - 提取 HOG 特征（`skimage.feature.hog`）
   - 训练 sklearn SVM（`SVC, kernel='rbf', decision_function_shape='ovr'`）
   - 保存模型到 `models/svm_char.pkl`
   - 输出训练集/验证集**字符级**准确率 + 混淆矩阵
2. 可选：`train_cnn.py` 训练轻量 CNN 作对比组

### Step 5：单图完整推理流水线

1. 编写 `pipeline.py`：
   - 输入：一张图片
   - YOLOv8-Pose 检测车牌区域 + 4 角点
   - 透视变换校正
   - 灰度化 → 自适应二值化 → 水平投影去上下边框 → 垂直投影切字符
   - **字符数校验**：切错数 → 标记识别失败，不硬凑
   - HOG 特征提取 → SVM 分类每个字符
   - 输出：识别的车牌号码
   - 可视化中间结果（检测框、角点、校正图、分割结果、识别结果）

### Step 6：视频实时识别模块（满足实验要求 2）

1. 编写 `video_recognition.py`：
   - 视频源：自拍一段真实视频（白天/傍晚、停车/行驶、正面/斜侧、远近各一段，车牌在画面中宽度 ≥ 80-100px）
   - `cv2.VideoCapture` 逐帧推理，复用 Step 5 单图流水线
   - **IoU 跟踪**：按检测框 IoU 关联同一辆车跨帧（或接 ByteTrack）
   - **跨帧多帧投票**：对同一车牌 ID 缓存最近 N 帧的逐字符识别结果，按位置做多数票融合，输出稳定车牌号
   - **实时性统计**：记录 FPS、单帧耗时
   - 输出：带标注的播放视频 + 识别结果日志
2. 保留识别失败/出错帧作为报告"问题分析"素材

### Step 7：分场景评测

1. 编写 `evaluate.py`，分两部分：
   - **CCPD 量化评测**：按子集（base, blur, weather, tilt, rotate, db, fn, challenge）分别评
     - 指标：检测召回率/精度、字符级准确率、**车牌级完全匹配率**
     - 生成评测结果表格和可视化图
   - **自拍视频定性评测**：统计视频识别成功率、实时 FPS、失败案例分析
2. 结果保存到 `results/`

## 复杂环境应对策略

| 复杂场景 | 应对方式 |
|----------|---------|
| 光照变化 | YOLOv8-Pose 数据增强 + 自适应二值化（Otsu/Gaussian） |
| 污损遮挡 | YOLOv8-Pose 训练含遮挡样本 + 分割字符数校验 |
| 运动模糊 | CCPD blur 子集参与训练 |
| 倾斜/透视畸变 | 4 角点透视变换校正 |
| 远近拍摄 | YOLOv8-Pose 多尺度检测 |
| 雨雪天气 | CCPD weather 子集参与训练 |
| 视频实时性 | 轻量 YOLOv8n + HOG+SVM 推理快 + 跨帧投票纠错 |
